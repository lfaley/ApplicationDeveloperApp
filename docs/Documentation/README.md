# Documentation

This folder consolidates all project documentation for efficient access and maintenance. Documentation is organized by topic and type:

## Structure

- `agentsPart2/` — Agent architecture, repo standards, and implementation plans
- `do_next/` — Desktop app migration guides and checklists
- `docs/` — General handoff patterns and documentation templates
- `DOCS_EXISTING_PROJECTS/` — Existing project assessment and improvement templates
- `DOCS_NEW_PROJECTS/` — New project quick start, onboarding, and update guides
- `FeatureSet3/` — Major feature set plans, checklists, and phase documentation
- `GUI/` — User guides, UI/UX implementation, and GUI standards
- `MajorPhase2/` — Architecture, onboarding, and process guidelines
- `PLANNING/` — Project planning templates, RACI, risk, and research docs
- `STANDARDS/` — Coding, API, testing, security, and design standards

## Usage
- Start here for any project documentation, onboarding, or standards reference.
- Each subfolder contains README and topic-specific guides.
- All links and references are updated for this new structure.
